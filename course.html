function generateTimeSlots(): string[] {
  const slots: string[] = [];

The QR Code Magic

How unique tokens enable quick, secure order pickup

What Is a QR Code?

A QR code is like a shortcut that your phone camera can read. Instead of typing a long web address, you point your camera at a square image and it instantly takes you to that page.

💡
Security Through Tokens

Each order gets a unique, random token (like a secret password). The QR code contains a URL with this token. When scanned, the system verifies the token matches an order and checks if it's expired.

QR Token Verification Code

Here's the actual server code that verifies QR tokens:

CODE 
// app/api/orders/verify-qr/route.ts
export async function GET(request) {
  const token = searchParams.get("token");
  
  // Find order with matching QR token
  const order = await prisma.order.findFirst({
    where: { qrToken: token },
    include: { items: { include: { product: true } } }
  });
  
  if (!order) return errorResponse("Invalid QR code");
  
  // Check if token expired
  if (order.qrTokenExpiresAt < new Date()) {
    return errorResponse("QR code expired");
  }
  
  // Must be READY status
  if (order.status !== "READY") {
    return errorResponse("Order not ready");
  }
  
  return Response.json(order);
}
PLAIN ENGLISH

This code runs on the server when someone scans a QR code

Extract the token from the URL (the part after ?token=)

Look up any order that has this exact QR token...

Also fetch the order items and product details together

If no order found, stop here and return an error

Check: is the expiration date in the past?

If expired, reject with a helpful error message

Check: is the order status READY for pickup?

If not ready yet, reject with status info

All checks passed! Return the order details

QR Code in Emails (CID Inline)

QR codes are embedded directly in emails so they display without external links:

💡
CID Attachment Technique

The QR code image is attached to the email with a Content-ID, then referenced inline. This ensures the image shows up immediately without the user having to click "load images" or rely on external hosting.

CODE 
// Generate QR code as data URL
const qrDataUrl = await QRCode.toDataURL(scanUrl, {
  width: 256,
  margin: 2
});

// Convert to base64 and attach
const base64Data = qrDataUrl.split(',')[1];
attachments: [{
  filename: 'qr-code.png',
  content: base64Data,
  encoding: 'base64',
  cid: 'qr-code-cid'  // Content ID
}]
PLAIN ENGLISH

Create a QR code image from the scan URL

Make it 256 pixels wide with a small margin

Data URLs start with "data:image/png;base64," — we remove that prefix

Attach the image data to the email...

Call it qr-code.png in the attachment

The actual image data (binary turned into text)

Tell the email system it's base64-encoded

Give it a unique ID so the HTML can reference it

    await transporter.sendMail({...});

PLAIN ENGLISH

The function tries to send email but doesn't block other operations

Create connection to Gmail

If email isn't set up, just log a warning and exit quietly

Don't let missing email config break order placement!

If ANY error happens while sending...

Log it for debugging but DON'T throw an error upward

The order still goes through even if email fails!

Database Schema Overview

The database schema uses Prisma to define the data structure:

CODE 
// prisma/schema.prisma
model Order {
  id          String   @id @default(uuid())
  orderNumber String   @unique  // ABC123
  status      String           // NEW/READY/COMPLETED
  guestName   String
  guestEmail  String
  total       Decimal
  qrToken     String?  @unique  // For QR pickup
  qrTokenExpiresAt DateTime?  // Expiration
  items       OrderItem[]  // Related items
  createdAt   DateTime @default(now())
}
PLAIN ENGLISH

An Order stores all information about a customer's purchase

Each order gets a unique ID automatically generated as a UUID

A human-readable order number like "ABC123" — no duplicates allowed

Status tracks progress: NEW → READY → COMPLETED

The customer's name and email for contact

Total amount with decimal precision for currency

Optional QR token (the ? means it can be empty)

When the QR code stops working (for security)

List of all items in this order — linked records

Auto-set to current time when order is created
