  </what-built>
  <how-to-verify>
Run through the full customer checkout flow on the live production URL:

**Step 1 — Place test order with Stripe payment:**
1. Open production URL in an incognito browser window
2. Browse products, add 1-2 items to cart
3. Go to cart, click "Proceed to Checkout"
4. Fill in contact info:
   - Name: "Test Customer"
   - Email: your personal email address (you'll check inbox in Step 3)
   - Phone: optional
5. Select "Pay Online" (Stripe)
6. On the Stripe Checkout page, use test card:
   - Card: `4242 4242 4242 4242`
   - Expiry: any future date (e.g., `12/29`)
   - CVC: any 3 digits (e.g., `123`)
7. Click "Pay"

**Step 2 — Verify Stripe Dashboard:**
1. Go to https://dashboard.stripe.com/payments
2. Confirm: new payment appears with status "Succeeded"
3. Go to Developers → Webhooks → select your endpoint
4. Confirm: `checkout.session.completed` event shows "Delivered" status (not "Failed")
5. If it shows "Failed": check Vercel function logs for the webhook error

**Step 3 — Verify email delivery:**
1. Check the email address you used in checkout (Step 1, item 4)
2. Confirm: order confirmation email arrives within 2 minutes
3. Check the subject line: must say "Order Confirmed #... — Essa Cafe" (not ShopSeeds)
4. Check if email landed in inbox vs spam folder
5. Check your ADMIN_EMAIL inbox
6. Confirm: admin notification email arrives with subject "New Order #... — Essa Cafe"

**Step 4 — Verify order in admin panel:**
1. Go to production URL + `/admin/orders`
2. Confirm: test order appears in the list with status "New"
3. Click the order to open detail view
4. Confirm: customer name, email, items, and total are correct
5. Change status to "Ready for Pickup"
6. Confirm: status updates without error

**Step 5 — Verify no errors in Vercel logs:**
1. Go to Vercel Dashboard → your project → Functions tab (or Logs)
2. Look through logs from the last 15 minutes
3. Confirm: no `console.error` lines for unexpected errors
4. Expected OK patterns: `Order created from webhook: ORD-...`, `Order confirmation email sent:`, `Admin notification email sent:`
5. NOT OK patterns: `Webhook signature verification failed`, `Error sending order confirmation email:`, `Failed to create order`
