  <what-to-do>
Set up Vercel + Neon production infrastructure. Follow these steps in order:

**Step 1 — Create Vercel project (D-01, D-02, D-04):**
1. Go to https://vercel.com/new
2. Import the `shop-seeds` GitHub repository
3. Framework preset: Next.js (auto-detected)
4. Leave Build Command blank (uses package.json "build" script: `prisma generate && prisma migrate deploy && next build`)
5. Click "Deploy" — this first deploy will FAIL (no env vars yet) — that is expected

**Step 2 — Provision Neon database (D-03):**
1. In the Vercel project dashboard, go to Storage → Add Store → Neon
2. Create new Neon database, choose the region closest to your users
3. Vercel automatically adds `DATABASE_URL` to your project environment variables (pooled connection string)
4. Confirm: Project Settings → Environment Variables shows `DATABASE_URL` set to `postgresql://...@pooler.neon.tech/...?sslmode=require`

**Step 3 — Set remaining environment variables (D-05):**
Go to Vercel → Project Settings → Environment Variables. Add ALL of these to the **Production** scope (and Preview scope for testing):

| Variable | Value | Source |
|----------|-------|--------|
| `AUTH_SECRET` | output of `openssl rand -hex 32` | Run that command in terminal |
| `AUTH_URL` | `https://<your-vercel-domain>.vercel.app` | Your Vercel project URL |
| `ADMIN_EMAIL` | `your-gmail@gmail.com` | Owner's Gmail address |
| `GMAIL_USER` | `your-gmail@gmail.com` | Same Gmail account |
| `GMAIL_APP_PASSWORD` | 16-char app password | Google Account → Security → 2-Step Verification → App passwords |
| `NEXT_PUBLIC_APP_URL` | `https://<your-vercel-domain>.vercel.app` | Same as AUTH_URL |
| `STRIPE_SECRET_KEY` | `sk_live_...` | Stripe Dashboard → API Keys → Secret key |
| `STRIPE_PUBLISHABLE_KEY` | `pk_live_...` | Stripe Dashboard → API Keys → Publishable key |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | `pk_live_...` | Same as above |
| `STRIPE_WEBHOOK_SECRET` | set this AFTER Step 4 | Leave blank for now |

**Step 4 — Register Stripe webhook:**
1. Go to https://dashboard.stripe.com/webhooks
2. Click "Add endpoint"
3. Endpoint URL: `https://<your-vercel-domain>.vercel.app/api/webhook`
4. Events: select `checkout.session.completed`
5. Click "Add endpoint"
6. Click the endpoint, copy the "Signing secret" (starts with `whsec_live_`)
7. Back in Vercel → Environment Variables → add `STRIPE_WEBHOOK_SECRET` = the signing secret

**Step 5 — Trigger a fresh deploy:**
1. Go to Vercel → Deployments → click the most recent deployment → "Redeploy"
2. Or push any change to the `main` branch
3. Watch the build logs — you should see:
   - `prisma generate` output
   - `All migrations have been successfully applied` (or "No pending migrations")
   - `next build` output completing successfully
4. Build status should show "Ready" (not "Error")
