## Key Changes

### prisma/schema.prisma
- Removed: `careInstructions String?`, `stockQuantity Int @default(0)`, `lowStockThreshold Int @default(5)`, `@@index([stockQuantity])`
- Added: `isAvailable Boolean @default(true)`
- `npx prisma generate` run successfully — client types updated
- `npx prisma validate` exits 0

### lib/validators.ts
- `productSchema` now validates `isAvailable: z.boolean().default(true)` and `isFeatured: z.boolean().default(false)`
- Removed: `careInstructions`, `stockQuantity`, `lowStockThreshold`
- `ProductFormData` type updated automatically via `z.infer<>`

### lib/orders.ts
- Step 1: `if (product.stockQuantity < item.quantity) throw INSUFFICIENT_STOCK` → `if (!product.isAvailable) throw ITEM_UNAVAILABLE`
- Step 4 (stock decrement loop) deleted entirely
- `Prisma.TransactionIsolationLevel.Serializable` preserved

### lib/cart-store.ts
- `CartItem.stockQuantity: number` → `CartItem.isAvailable: boolean`
- `addItem`: `Math.min(existing.quantity + 1, product.stockQuantity)` → `existing.quantity + 1`
- `updateQuantity`: stock-cap block removed, `cappedQty = quantity` directly
- Storage key: `"shop-seeds-cart"` → `"essa-cafe-cart"`

### UI consumers (auto-fixed, Rule 1)
All files referencing the removed fields updated to use `isAvailable`:
- `StockBadge`: two states (Available / Unavailable) instead of three
- `ProductCard`, `AddToCartButton`: pass `isAvailable` to cart store
- `CartItem`: removed `max` attr and `disabled` from quantity controls
- `FeaturedProducts`, `ProductGrid`, `RelatedProducts`, `ShopPageClient`: Product interface updated
- Store pages: `stockQuantity: { gt: 0 }` filters → `isAvailable: true`
- Admin dashboard: counts unavailable products via `!p.isAvailable`
- Admin products API: `create` uses `isAvailable`/`isFeatured`

### prisma/seed.ts
Rewritten with coffee shop products: espresso drinks (Flat White, Cappuccino), filter coffee (Batch Brew, Cold Brew), food (Almond Croissant, Banana Bread). One product (`Banana Bread`) seeded as `isAvailable: false` to test the unavailable state.

## Database Migration Note

The Prisma schema is correct and the client is regenerated. The live database still has the old columns (`stockQuantity`, `careInstructions`, `lowStockThreshold`) and is missing `isAvailable`.

To apply the migration, run one of the following (dev only — all data will be lost on reset):

```bash
# Option 1: Reset and re-migrate (drops all data)
PRISMA_USER_CONSENT_FOR_DANGEROUS_AI_ACTION="yes" npx prisma migrate reset

# Option 2: Push schema directly without migration history (dev only)
npx prisma db push

# Option 3: Create migration file and apply manually
npx prisma migrate dev --name coffee_pivot_schema
# (will prompt for reset confirmation)
```
