### Pillar 4: Typography (3/4)

**Font sizes in use:**
- `text-xs` (12px) — 5 instances
- `text-sm` (14px) — 108 instances (body text, small labels)
- `text-base` (16px) — 1 instance
- `text-lg` (18px) — 10 instances (secondary headings)
- `text-xl` (20px) — 6 instances
- `text-2xl` (24px) — 6 instances (section headers)
- `text-3xl` (30px) — 8 instances
- `text-4xl` (36px) — 1 instance
- `text-5xl` (48px) — 1 instance (hero title)

**Font weights in use:**
- `font-medium` (500) — 74 instances
- `font-semibold` (600) — 65 instances
- `font-bold` (700) — 50 instances

**UI-SPEC contract specifies:**
- Only 4 sizes: 14px body, 14px label (600), 18px h3, 24px h2, 32px h1
- Only 2 weights: 400 (regular) and 600 (semibold)

**Deviation analysis:**
- **Font-weight violation:** Code uses 500 (medium) and 700 (bold) extensively, but spec requires only 400 and 600
- **Font-size violation:** 9 distinct sizes used (xs through 5xl) vs. spec's 4 declared sizes
- Headings correctly use bold weights, but body text inconsistently mixes font-medium with font-semibold for labels
- Hero title correctly renders at text-5xl (48px ≈ 32px spec with md: breakpoint scaling)

**Assessment:** Spec requires consolidation to exactly 2 weights and 4 sizes. Current implementation uses 3 weights and 9 sizes. The extra sizes aren't *wrong* but exceed the intentional constraints of the design system.
