The 10% accent color (`forest-900`) is **reserved for:**

## Component Spec: Key Changes in Phase 01.1

### 1. ProductForm Changes

**Fields removed:**
- Stock Quantity input (was numeric field, 0-999 range)
- Low Stock Threshold input (was numeric field, 0-999 range)
- Care Instructions editor (was Tiptap rich-text field)

**Fields added:**
- `isAvailable` toggle (boolean checkbox, default: checked/on)
- `isFeatured` toggle (boolean checkbox, was missing from form, default: unchecked)

**Field order:**
1. Product Name (required)
2. Category (required dropdown)
3. Price (required numeric, $0.01+)
4. Description (Tiptap rich-text editor)
5. Images (up to 5 URLs)
6. **`isAvailable` toggle** (new, default on)
7. **`isFeatured` toggle** (new, default off)

**Labels:**
- `isAvailable`: "Available for purchase" or "Make available"
- `isFeatured`: "Feature on homepage"

### 2. DashboardStats Card Layout

**Cards removed:**
- "Low Stock Alerts" card (amber-tinted)

**Cards remaining (2-column grid):**
- "New Orders" card (blue-tinted, shows integer count)
- "Today's Revenue" card (forest-tinted, shows formatted price)

**Layout:**
- Desktop: 2 equal-width columns
- Mobile: Stack to full-width (1 column)

## Availability Badge (Formerly StockBadge)

Replaces the 3-state badge (In Stock / Low Stock / Out of Stock) with a 2-state badge.

| State | Display Text | Background Color | Text Color |
|-------|--------------|------------------|-----------|
| Available | "Available" | `sage-100` | `sage-500` |
| Unavailable | "Unavailable" | `forest-100` | `forest-500` |

**Usage:** Product detail page (replaces StockBadge), product cards (if visible).

**Interaction:** Read-only badge on storefront. Admin controls state via `isAvailable` toggle in ProductForm.
