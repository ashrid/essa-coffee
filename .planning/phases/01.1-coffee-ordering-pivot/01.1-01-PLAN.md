    ```typescript
    export const productSchema = z.object({
      name: z.string().min(2).max(200),
      description: z.string().optional(),
      price: z.number().positive().multipleOf(0.01),
      isAvailable: z.boolean().default(true),
      isFeatured: z.boolean().default(false),
      categoryId: z.string().cuid(),

    });
    ```

    **lib/orders.ts** — Simplify createOrderAtomically:

    REPLACE Step 1 (lines 25-33):
    ```typescript
    // OLD — delete this:
    for (const item of data.items) {
      const product = await tx.product.findUniqueOrThrow({
        where: { id: item.productId },
      });
      if (product.stockQuantity < item.quantity) {
        throw new Error(`INSUFFICIENT_STOCK:${product.name}`);
      }
    }
    ```
    WITH:
    ```typescript
    // NEW — availability check:
    for (const item of data.items) {
      const product = await tx.product.findUniqueOrThrow({
        where: { id: item.productId },
      });
      if (!product.isAvailable) {
        throw new Error(`ITEM_UNAVAILABLE:${product.name}`);
      }
    }
    ```

    DELETE Step 4 entirely (the stock decrement loop, approximately lines 59-65):
    ```typescript
    // DELETE THIS ENTIRE BLOCK:
    // 4. Deduct stock atomically
    for (const item of data.items) {
      await tx.product.update({
        where: { id: item.productId },
        data: { stockQuantity: { decrement: item.quantity } },
      });
    }
    ```

    Steps 2 (calculate total) and 3 (create order) remain unchanged.

    The transaction isolation level `Prisma.TransactionIsolationLevel.Serializable` stays unchanged.
