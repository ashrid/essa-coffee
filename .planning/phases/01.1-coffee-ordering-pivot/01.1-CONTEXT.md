# Phase 1.1: Coffee Ordering Pivot - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Adapt the existing ShopSeeds store for coffee shop ordering under the brand "Essa Cafe". Remove all plant/seed product content and stock-quantity management. Replace with a coffee menu structure. Ensure the full order flow (browse drinks → cart → checkout → order creation) works correctly. Deployment and launch are out of scope.

</domain>

<decisions>
## Implementation Decisions

### Schema migration
- **D-01:** Full Prisma migration — drop `careInstructions`, `stockQuantity`, `lowStockThreshold` columns and the `@@index([stockQuantity])` index from the `Product` model
- **D-02:** Add `isAvailable Boolean @default(true)` to `Product` — replaces stock-based availability checks everywhere
- **D-03:** Keep `isFeatured Boolean` — manual admin toggle, no change to field semantics

### Availability model
- **D-04:** All `stockQuantity > 0` checks replaced by `isAvailable = true` — in queries, API routes, cart store, and checkout transaction
- **D-05:** Checkout transaction (`lib/orders.ts`): replace stock check with `!product.isAvailable` guard, remove the stock-decrement step entirely. Serializable isolation level can be kept or downgraded to ReadCommitted — no concurrent decrement risk remains
- **D-06:** `CartItem.stockQuantity: number` in cart store → `isAvailable: boolean` — cart no longer enforces quantity limits
- **D-07:** `StockBadge` component repurposed/renamed to show "Available" / "Unavailable" based on `isAvailable`

### Brand identity
- **D-08:** Shop name: **Essa Cafe** — replaces "ShopSeeds" in all UI text, metadata, footer, admin panel, email templates, and page titles
- **D-09:** Tagline (Claude-authored placeholder): **"Order ahead, pick up fresh"** — used in homepage hero and `<meta>` description
- **D-10:** Visual palette: update `tailwind.config.ts` hex values for `forest` → espresso brown and `sage` → warm caramel/amber. Class names (`forest-*`, `sage-*`) remain unchanged across all components — zero component edits for color

  Proposed palette values (Claude's discretion to finalize exact shades):
  - `forest-600` → espresso `#3B1F0E` (primary brand, currently used for hero bg, buttons, nav)
  - `forest-900` → dark roast `#1A0D06`
  - `forest-500` → medium roast `#5C3317`
  - `forest-50`/`100` → warm parchment tints
  - `sage-*` → warm amber/caramel tones replacing green secondary

### Menu structure
- **D-11:** Four categories replace Houseplants/Seeds/Succulents: **Hot Drinks, Iced Drinks, Specialty, Food**
- **D-12:** Seed data (`prisma/seed.ts`) updated with coffee menu items — a handful of demo drinks per category so the admin panel and storefront are usable out of the box
- **D-13:** Existing plant product data wiped and replaced by coffee seed data

### Admin panel
- **D-14:** `ProductForm` removes: Stock Quantity field, Low Stock Threshold field, Care Instructions editor
- **D-15:** `ProductForm` adds: `isAvailable` toggle (boolean checkbox, default on) and `isFeatured` checkbox (was missing from the form — admin needs explicit control now that stock is gone)
- **D-16:** Admin dashboard removes "Low Stock Alerts" stat card → 2-stat layout: **New Orders** + **Today's Revenue**
- **D-17:** `DashboardStats` component props change: remove `lowStockCount`, update layout to 2 columns

### Branding copy sweep
- **D-18:** All occurrences of "ShopSeeds", "plants", "seeds", "succulents", "houseplants", "care instructions", "growing instructions", "stock", "inventory" replaced with coffee-appropriate language across store and admin UI
- **D-19:** Product detail page removes the "Care & Growing Instructions" section entirely
- **D-20:** Homepage hero updated: title "Fresh Coffee, Ready for Pickup" (or similar), description uses tagline from D-09

### Claude's Discretion
- Exact espresso/caramel hex values for the palette
- Demo product names and descriptions in seed data
- 2-stat dashboard layout (side-by-side vs stacked)
- AvailabilityBadge visual styling (color, wording)

</decisions>

<specifics>
## Specific Ideas

- Keep the green/earthy *feel* of the palette (warm and natural) but shift it from plant-green to coffee-brown — Essa Cafe should still feel organic and approachable, not a generic corporate coffee chain
- Admin should be able to mark a drink "unavailable" for the day (e.g., oat milk ran out) via the isAvailable toggle

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `createOrderAtomically` in `lib/orders.ts`: keep the transaction pattern, simplify by removing the decrement loop (steps 1 + 4 change, steps 2 + 3 stay identical)
- `TiptapEditor` in `ProductForm.tsx`: still used for product description — only the Care Instructions instance is removed
- `DashboardStats`: component is easy to refactor — just remove one prop and update the grid from 3 to 2 columns
- `prisma/seed.ts`: full rewrite — existing plant products out, coffee products in

### Established Patterns
- Color class naming (`forest-*`, `sage-*`, `cream-*`): stays unchanged — only hex values in `tailwind.config.ts` change
- `isAvailable` queries follow the same pattern as current `stockQuantity: { gt: 0 }` filters — simple find/replace in Prisma where clauses
- `productSchema` in `lib/validators.ts`: remove 3 fields, add `isAvailable: z.boolean().default(true)` — downstream `ProductFormData` type updates automatically

### Integration Points
- Checkout API (`app/api/checkout/route.ts`) calls `createOrderAtomically` — no changes needed there if `lib/orders.ts` signature is preserved
- Cart store `CartItem` type is referenced in `CartDrawer`, `CartItem` component, and `AddToCartButton` — all need `stockQuantity` → `isAvailable` update

</code_context>
