**Gathered:** 2026-03-25
**Status:** Ready for planning
**Source:** User request via `/gsd:do`

---

<domain>
## Phase Boundary

Add in-page QR code scanning capability to the admin/scan page for mobile devices, allowing employees to scan order QR codes directly without navigating between pages or using external apps.

This is a mobile-first enhancement to streamline the order pickup workflow for staff members.

</domain>

<decisions>
## Implementation Decisions

### QR Scanning
- In-page camera access via browser APIs (not external apps)
- Real-time QR detection and decoding
- Must work on mobile browsers (Chrome, Safari, Samsung Internet)

### User Flow
- Employee opens admin/scan on mobile device
- Camera view appears directly in page
- Scan QR code → order details display immediately
- One-tap reset for continuous scanning mode
- No page navigation/jumps during the flow

### Fallback
- Manual order number entry must still work if camera denied/unavailable

### Technical Approach
- HTML5 getUserMedia API for camera access
- QR code scanning library (zxing-js, jsqr, or similar)
- React component for camera integration
- Keep existing order lookup functionality as fallback

</decisions>

<specifics>
## Specific Ideas

1. **Camera Component**: Create a `QRScanner` component that wraps camera access
2. **Auto-scan**: Continuous scanning mode that doesn't require manual button press
3. **Success Feedback**: Visual/audio feedback when QR is successfully scanned
4. **Error Handling**: Clear messages for permission denied, no camera, invalid QR
5. **Mobile Optimization**: Full-screen camera view on mobile, touch-friendly controls

</specifics>

<deferred>
## Deferred Ideas

- Desktop camera support (mobile-first for this phase)
- Batch scanning (scan multiple orders at once)
- Barcode support (only QR codes needed for orders)

</deferred>
