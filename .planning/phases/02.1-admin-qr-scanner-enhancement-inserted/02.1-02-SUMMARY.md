# Phase 02.1 Plan 02: QR Scanner Components Summary

## One-Liner
Created QRScanner and ScannerContainer components with html5-qrcode integration for in-page camera-based QR scanning with SSR safety, permission management, and mobile optimization.

## What Was Built

### QRScanner.tsx (175 lines)
Full-featured QR code scanner component with:
- **html5-qrcode integration**: Uses Html5QrcodeScanner with QR_CODE format support
- **Mobile optimization**: 2 FPS (vs default 10) for battery life
- **Permission handling**: Detects and displays states for granted, denied, prompt, and unsupported
- **Token parsing**: Supports both full URLs (`?token=xxx`) and raw token formats
- **Error filtering**: Ignores normal scanning errors ("No MultiFormat Readers")
- **Camera cleanup**: useEffect cleanup calls `scanner.clear()` to release camera
- **SSR safety**: Checks `typeof window === 'undefined'` before browser API access
- **UI states**: Loading, scanning, permission-denied, unsupported, and error states with project styling

### ScannerContainer.tsx (41 lines)
SSR-safe wrapper component with:
- **Dynamic import**: `ssr: false` prevents server-side rendering errors
- **Suspense fallback**: Loading spinner with project colors during import
- **Props forwarding**: Passes `onScan` and `onError` callbacks to QRScanner
- **Clean separation**: Container handles loading, QRScanner handles scanning

## Key Implementation Details

### Camera Configuration
```typescript
{
  fps: 2,  // Mobile battery optimization
  qrbox: { width: 250, height: 250 },
  aspectRatio: 1.0,
  showTorchButtonIfSupported: true,
  showZoomSliderIfSupported: true,
  formatsToSupport: [Html5QrcodeSupportedFormats.QR_CODE],
}
```

### Token Parsing Logic
The scanner handles multiple QR code formats:
1. Full URL with token: `https://.../admin/scan?token=xxx`
2. URL without token param (attempts to extract anyway)
3. Raw token string (uses directly)

### Error Handling Strategy
- Permission denied: Shows amber warning with retry button
- Unsupported browser: Shows gray info state
- Scanner errors: Shows red error state with message
- Normal scanning: "No MultiFormat Readers" filtered silently
