# Phase 02.1: Admin QR Scanner Enhancement - Research

**Researched:** 2026-03-25
**Domain:** QR Code Scanning in React/Next.js Mobile Web Apps
**Confidence:** HIGH

## Summary

This phase requires adding in-page QR code scanning capability to the existing admin/scan page. The current implementation relies on external QR scanning (customers show QR, staff use phone camera to scan URL with token). The enhancement adds direct camera access within the web app for a streamlined workflow.

Key findings:
1. **html5-qrcode** (v2.3.8) is the most suitable library - battle-tested, all-in-one solution with built-in camera handling and UI
2. **Mobile browser support is excellent** in 2024 - all major mobile browsers (Chrome Android, Safari iOS 11+, Samsung Internet) support getUserMedia
3. **HTTPS is mandatory** - camera access requires secure context
4. **Next.js requires dynamic imports with `ssr: false`** - camera APIs are browser-only
5. **Permission handling is critical** - must gracefully handle denied permissions with fallback to manual entry

**Primary recommendation:** Use `html5-qrcode` v2.3.8 with dynamic import in a client component, with comprehensive error handling and manual order lookup as fallback.

**What:** Use Next.js dynamic import to load scanner only on client
**When to use:** Required for any browser-only APIs (camera, getUserMedia)
**Example:**

### Pattern 2: html5-qrcode Integration
**What:** Use Html5QrcodeScanner class for built-in UI
**When to use:** When you want minimal custom UI work
**Example:**
```typescript
// Source: https://github.com/scanapp-org/html5-qrcode-react
'use client';

import { useEffect, useRef } from 'react';
import { Html5QrcodeScanner } from 'html5-qrcode';

interface QRScannerProps {
  onScan: (token: string) => void;
  onError?: (error: string) => void;
}

export function QRScanner({ onScan, onError }: QRScannerProps) {
  const scannerRef = useRef<Html5QrcodeScanner | null>(null);
  const elementId = 'qr-scanner-region';

  useEffect(() => {
    // Only run on client
    if (typeof window === 'undefined') return;

    scannerRef.current = new Html5QrcodeScanner(
      elementId,
      {
        fps: 2,                    // Lower FPS for mobile battery
        qrbox: { width: 250, height: 250 },
        aspectRatio: 1.0,          // Square viewfinder
        showTorchButtonIfSupported: true,
        showZoomSliderIfSupported: true,
      },
      false                       // verbose
    );

    scannerRef.current.render(
      (decodedText) => {
        // Success callback
        onScan(decodedText);
      },
      (errorMessage) => {
        // Error callback - called frequently during scanning
        // Only report critical errors, not "no QR found"
        if (!errorMessage?.includes('No MultiFormat Readers')) {
          onError?.(errorMessage);
        }
      }
    );

    return () => {
      scannerRef.current?.clear().catch(console.error);
    };
  }, [onScan, onError]);

  return <div id={elementId} className="w-full" />;
}
```

### Pattern 3: Permission-Aware Scanner Flow
**What:** Check permissions before initializing scanner, provide clear fallback
**When to use:** Required for good UX on mobile
**Example:**
```typescript
// Permission check pattern
async function checkCameraPermission(): Promise<PermissionState> {
  try {
    // Check if we can get camera access
    const stream = await navigator.mediaDevices.getUserMedia({ video: true });
    stream.getTracks().forEach(track => track.stop());
    return 'granted';
  } catch (err) {
    if (err instanceof DOMException) {
      if (err.name === 'NotAllowedError') return 'denied';
      if (err.name === 'NotFoundError') return 'not-found';
    }
    return 'prompt';
  }
}
```

### Anti-Patterns to Avoid
- **Don't use `navigator.mediaDevices` without feature detection:** Always check `typeof window !== 'undefined'` and `navigator.mediaDevices?.getUserMedia`
- **Don't forget cleanup:** Always call `scanner.clear()` in useEffect cleanup to release camera
- **Don't scan at high FPS on mobile:** 10+ FPS drains battery; use 2-5 FPS for mobile
- **Don't block the main thread:** html5-qrcode handles this, but custom implementations should use Web Workers

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Camera enumeration | Custom getUserMedia calls | html5-qrcode's built-in camera selection | Handles iOS/Android differences, permission flows, fallback cameras |
| QR decoding algorithm | Custom QR parser | html5-qrcode's decoder | QR codes have complex error correction, multiple modes, timing patterns |
| Camera preview UI | Custom video element styling | html5-qrcode's built-in UI | Includes viewfinder, scan region highlighting, torch/zoom controls |
| Permission retry logic | Custom permission prompts | Browser-native flows with clear fallback UI | Users trust browser prompts more than custom ones |

**Key insight:** Camera access and QR decoding involve complex cross-browser compatibility, permission handling, and error correction mathematics. A battle-tested library saves weeks of development and handles edge cases you will not discover until production.

## Common Pitfalls

### Pitfall 1: SSR/Hydration Errors
**What goes wrong:** Importing html5-qrcode directly causes "window is not defined" errors during Next.js SSR
**Why it happens:** The library accesses browser APIs immediately on import
**How to avoid:** Always use dynamic import with `ssr: false`
**Warning signs:** Build fails with ReferenceError to window/document/navigator

### Pitfall 2: Camera Permission Denied Without Fallback
**What goes wrong:** User denies camera permission and has no way to complete the task
**Why it happens:** No alternative path provided
**How to avoid:** Always show manual order entry form alongside scanner; detect permission state and show helpful guidance
**Warning signs:** User gets stuck on scanner page with no options

### Pitfall 3: Scanner Not Cleaning Up
**What goes wrong:** Camera stays active after leaving page, causing battery drain and "camera in use" errors
**Why it happens:** Missing cleanup in useEffect return function
**How to avoid:** Always call `scanner.clear()` in cleanup; consider cleanup on route change
**Warning signs:** Camera LED stays on after navigating away; subsequent scans fail

### Pitfall 4: HTTPS Requirement in Development
**What goes wrong:** Scanner works on localhost but fails in production on HTTP
**Why it happens:** getUserMedia requires secure context (HTTPS or localhost)
**How to avoid:** Always test on HTTPS; use ngrok or similar for mobile testing
**Warning signs:** `NotAllowedError` or `NotSupportedError` in production only

### Pitfall 5: iOS WKWebView Limitations
**What goes wrong:** Scanner works in Safari but not in Chrome/Firefox on iOS
**Why it happens:** Third-party browsers on iOS use WKWebView which had limited getUserMedia support before iOS 14.3
**How to avoid:** Document iOS 14.3+ requirement; test in Safari primarily
**Warning signs:** Scanner fails specifically in Chrome iOS but works in Safari

### Pitfall 6: High FPS Draining Battery
**What goes wrong:** Mobile devices heat up and battery drains quickly during scanning
**Why it happens:** Default 10 FPS is too high for continuous mobile use
**How to avoid:** Use 2-5 FPS for mobile scanning scenarios
**Warning signs:** Device gets warm, battery percentage drops rapidly

## Code Examples

### Complete Scanner Component with Error Handling
```typescript
// app/(admin)/admin/scan/QRScanner.tsx
'use client';

import { useEffect, useRef, useState, useCallback } from 'react';
import { Html5QrcodeScanner, Html5QrcodeSupportedFormats } from 'html5-qrcode';
import { AlertCircle, Camera, CameraOff } from 'lucide-react';
import { Button } from '@/components/ui/button';

interface QRScannerProps {
  onScan: (token: string) => void;
  onError?: (error: string) => void;
}

type ScannerState = 'initializing' | 'scanning' | 'error' | 'permission-denied';

export function QRScanner({ onScan, onError }: QRScannerProps) {
  const scannerRef = useRef<Html5QrcodeScanner | null>(null);
  const [state, setState] = useState<ScannerState>('initializing');
  const [errorMessage, setErrorMessage] = useState<string>('');
  const elementId = 'admin-qr-scanner';

  const handleScan = useCallback((decodedText: string) => {
    // Validate it's a QR token URL from our domain
    try {
      const url = new URL(decodedText);
      const token = url.searchParams.get('token');
      if (token && url.pathname.includes('/admin/scan')) {
        onScan(token);
      } else {
        onError?.('Invalid QR code. Please scan an order pickup code.');
      }
    } catch {
      onError?.('Invalid QR code format.');
    }
  }, [onScan, onError]);

  useEffect(() => {
    if (typeof window === 'undefined') return;

    // Check for camera support
    if (!navigator.mediaDevices?.getUserMedia) {
      setState('error');
      setErrorMessage('Camera access is not supported in this browser.');
      return;
    }

    // Check permission before initializing
    navigator.permissions?.query({ name: 'camera' as PermissionName })
      .then(permissionStatus => {
        if (permissionStatus.state === 'denied') {
          setState('permission-denied');
          return;
        }
        initializeScanner();
      })
      .catch(() => {
        // permissions API not supported, try anyway
        initializeScanner();
      });

    function initializeScanner() {
      scannerRef.current = new Html5QrcodeScanner(
        elementId,
        {
          fps: 2,  // Lower for mobile battery life
          qrbox: { width: 250, height: 250 },
          aspectRatio: 1.0,
          showTorchButtonIfSupported: true,
          formatsToSupport: [Html5QrcodeSupportedFormats.QR_CODE],
        },
        false
      );

      scannerRef.current.render(
        handleScan,
        (error) => {
          // Ignore "no QR found" errors - they happen constantly while scanning
          if (error?.includes('No MultiFormat Readers')) return;

          // Handle permission errors
          if (error?.includes('NotAllowedError')) {
            setState('permission-denied');
          }
        }
      ).then(() => {
        setState('scanning');
      }).catch((err) => {
        setState('error');
        setErrorMessage(err.message || 'Failed to start camera');
        onError?.(err.message);
      });
    }

    return () => {
      scannerRef.current?.clear().catch(console.error);
    };
  }, [handleScan, onError]);

  if (state === 'permission-denied') {
    return (
      <div className="text-center p-6">
        <CameraOff className="w-12 h-12 text-amber-500 mx-auto mb-4" />
        <h3 className="font-semibold text-forest-900 mb-2">Camera Access Denied</h3>
        <p className="text-forest-600 text-sm mb-4">
          Please enable camera access in your browser settings, or use manual order entry below.
        </p>
        <Button onClick={() => window.location.reload()} variant="outline">
          Try Again
        </Button>
      </div>
    );
  }

  if (state === 'error') {
    return (
      <div className="text-center p-6">
        <AlertCircle className="w-12 h-12 text-red-500 mx-auto mb-4" />
        <h3 className="font-semibold text-forest-900 mb-2">Scanner Error</h3>
        <p className="text-forest-600 text-sm">{errorMessage}</p>
      </div>
    );
  }

  return (
    <div className="relative">
      {state === 'initializing' && (
        <div className="absolute inset-0 flex items-center justify-center bg-cream-50">
          <Camera className="w-8 h-8 text-forest-600 animate-pulse" />
        </div>
      )}
      <div id={elementId} className="w-full" />
    </div>
  );
}
```

## Open Questions

1. **Should we support file upload fallback for desktop?**
   - What we know: html5-qrcode supports file scanning; desktop camera is deferred
   - What's unclear: Whether to add "upload QR image" button for desktop users
   - Recommendation: Skip for this phase (mobile-first), add if requested later

2. **What is the QR token format?**
   - What we know: Tokens are passed as `?token=xxx` query param to `/admin/scan`
   - What's unclear: Whether QR codes contain full URLs or just tokens
   - Recommendation: Support both - check if scanned text is URL (parse token) or raw token (use directly)

3. **Should we auto-redirect after successful scan?**
   - What we know: Current flow shows order details on same page after token verification
   - What's unclear: Whether scanner should stay active for batch scanning
   - Recommendation: Stop scanner on successful scan, show order details with "Scan Next" button
